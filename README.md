本プロジェクトは、`Sale Report.csv`, `Amazon Sale Report.csv`, `International sale Report.csv`, `May-2022.csv` の複数のデータソースを統合し、販売チャネル、製品カテゴリ、サイズ、および製品スタイルごとの売上パフォーマンスを分析することを目的としています。主要なトレンドの特定、懸念事項の指摘、および売上と収益性を向上させるための具体的な戦略の提案を行います。

### 使用データ

*   **Sale Report.csv**: 製品在庫と販売情報。
*   **Amazon Sale Report.csv**: Amazonチャネルからの販売トランザクションデータ。
*   **International sale Report.csv**: 海外販売チャネルからの販売トランザクションデータ。
*   **May-2022.csv**: 製品マスターデータ、価格情報、重量など。

### データ準備と統合

1.  **データ読み込み**: すべてのCSVファイルをPandas DataFrameとして読み込みました。
2.  **欠損値処理**: 各DataFrameの欠損値を確認し、数値型は中央値、カテゴリ型は最頻値で補完しました。特に、`Amazon Sale Report` では欠損値の多い列を削除しました。`International sale Report` と `May-2022` のデータ型を変換し、欠損値を適切に処理しました。
3.  **列名統一**: 複数のDataFrame間で共通のキーとなる列（`SKU Code` を `SKU`、`Design No.` や `Style Id` を `Style`、`DATE` を `Date` など）の名称を統一しました。
4.  **データ統合**: 
    *   `Amazon Sale Report` と `International sale Report` を結合し、`Sales Channel` 列を追加して `Combined Sales` DataFrame を作成しました。
    *   `Sale Report` と `May-2022` を結合し、`Product Master` DataFrame を作成しました。この際、`May-2022` の情報を優先し、重複する列はリネームまたは削除しました。
    *   `Combined Sales` と `Product Master` を結合し、`Final Integrated Data` DataFrame を作成しました。この最終統合データからは、結合によって生じた100%欠損値の列を削除し、分析に適した状態にしました。

### 実施した分析

*   **販売チャネル別売上分析**: 各販売チャネル（Amazon, International）の総売上を棒グラフで可視化しました。
*   **サイズ別売上分析**: 各サイズカテゴリごとの総売上を棒グラフで可視化し、上位/下位のサイズを特定しました。
*   **トップスタイル売上分析**: 総売上に基づいてトップ10の`Style`コードを特定し、その売上を棒グラフで可視化しました。
*   **SKU/Style別売上パフォーマンス分析**: `SKU`と`Style`ごとの総販売数量、総売上金額、販売頻度を計算し、上位/下位の製品を特定しました。
*   **時系列売上予測**: 各販売チャネルの月次総売上金額の時系列データを作成し、SARIMAモデルを訓練して将来の売上を予測しました。`International`チャネルについては予測を実行し可視化しましたが、`Amazon`チャネルはデータ期間が短すぎるためスキップされました。

### 主要な分析結果と洞察

*   **販売チャネル**: `Amazon`が主要な売上貢献チャネルであり、`International`も一定の売上を上げています。ただし、`Amazon`チャネルのデータは時系列分析には不十分でした。
*   **製品パフォーマンス**: 特定の`SKU`と`Style`が売上の大部分を占めており、上位製品と下位製品の間に大きな乖離が見られます。サイズ別では、最も売上の高いサイズと低いサイズが特定されました。
*   **時系列予測 (`International`チャネル)**: SARIMAモデルは`International`チャネルの将来3ヶ月間の売上を予測しましたが、過去の変動パターンを反映しつつも、明確な長期的トレンドは示されませんでした。予測の信頼性は過去の急激な変動や外部要因に影響されます。
*   **データギャップ**: `Product Master` DataFrameとの統合後、多くの製品属性列（`Category`, `Weight`, `MRP`関連、`Color`など）が`Final Integrated Data`で100%欠損値となりました。これは、結合キーの不一致や元のデータソースに情報がなかったことが原因であり、結果として製品カテゴリごとの分析や利益計算が現在のデータセットでは不可能であるという課題が残っています。

### 提言と次のステップ

1.  **データ収集の優先順位付け**: 
    *   **`Amazon`チャネル**: 少なくとも12ヶ月以上の過去データを継続的に収集し、より信頼性の高い時系列予測モデルを構築できるようにすることが最優先です。
    *   **製品属性データ**: 製品カテゴリ情報と原価情報（コストデータ）は、真の利益分析と収益性に基づく戦略的意思決定に不可欠です。これらのデータの緊急な収集と統合を強く推奨します。
2.  **`International`チャネルの変動要因分析**: `International`チャネルの過去の売上変動（急激な上昇や下降）の背後にある要因（特定のキャンペーン、市場イベント、競合動向、為替レートなど）を詳細に調査し、予測の精度向上と戦略的なプロモーション計画に役立てます。
3.  **製品戦略の最適化**: SKU/Style別パフォーマンス分析で特定されたトップ製品の成功要因を深く掘り下げ、その知見を他の製品に適用することを検討します。パフォーマンスの低い製品については、在庫の最適化、プロモーションの強化、または段階的な廃止を検討します。
4.  **利益に基づく意思決定への移行**: コストデータが利用可能になり次第、売上だけでなく製品ごとの利益率を評価する分析に移行し、最も収益性の高い製品やチャネルにリソースを再配分することを推奨します。
